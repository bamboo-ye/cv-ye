项目概述

项目背景

趣斗图是一个面向网络社交场景的表情包合成平台。项目面临的核心挑战包括：算法服务耗时高、资源有限易因突发流量过载不可用、用户离线无法接收结果。作为服务端研发工程师，我主导突破高可用与消息可靠性难题，最终实现系统稳定支撑全业务场景。

效果演示

核心工作

Kafka消息队列 + 动态降级方案

  • 设计任务消息存储至 Kafka，实现生产与消费解耦
  • 算法生成模块根据负载动态决策任务消费速率
  • 消费者生成结果后通过消息队列推送至用户端
  • 成功降低算法服务过载率，保障高峰期服务可用性

基于 Redis Cluster 的混合推送架构

  • 利用 Redis Pub/Sub 机制实现在线用户消息实时投递
  • 针对用户离线场景，同步将消息缓存至 Redis
  • 用户重新上线后自动触发未读消息拉取，避免消息丢失
  • 结合 WebSocket 实现毫秒级消息触达
